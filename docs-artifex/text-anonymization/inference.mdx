        # `text_anonymization()`

        Perform inference with a Text Anonymization model, on a single input string or a list of input 
		strings. For each input string, this will return an anonymized version of the string, in
		which all Personally Identifiable Information (PII) have been replaced with a placeholder.

		If no model is loaded, the base Text Anonymization model [tanaos/tanaos-text-anonymizer-v1](https://huggingface.co/tanaos/tanaos-text-anonymizer-v1) 
		will be used by default. For more information on what Personal Identifiable Information 
		(PII) the base Text Anonymization model is trained to extract, see the [model's Hugging Face page](https://huggingface.co/tanaos/tanaos-text-anonymizer-v1).

		<APIDescriptionSection title='Arguments'>
			- **text** <ParameterTypeBadge>str | list[str]</ParameterTypeBadge>\
                A string or a list of strings to remove Personally Identifiable Information (PII) 
				from. The model will return a version of the input text(s) in which all 
				detected PII have been replaced with a placeholder.
			- **entities_to_mask** <ParameterTypeBadge>list[str]</ParameterTypeBadge> <OptionalParameterBadge>optional</OptionalParameterBadge>\
				A list of entity types to mask in the input text(s). If not provided, all 
				supported entity types will be masked. Supported entity types are 
				`"PERSON"`, `"LOCATION"`, `"DATE"`, `"ADDRESS"` and `"PHONE_NUMBER"`.
			- **mask_token** <ParameterTypeBadge>str</ParameterTypeBadge> <OptionalParameterBadge>optional</OptionalParameterBadge> <DefaultParameterBadge value="'[MASKED]'"/>\
				The placeholder string to use for masking detected PII. Default is `"[MASKED]"`.
			- **device** <ParameterTypeBadge>int</ParameterTypeBadge> <OptionalParameterBadge />\
				A string which specifies the device to use. It can be one of the following:
				- `0` - Use the first GPU available
				- `1` - Use the second GPU available
				- ...
				- `-1` - Use the CPU
				
				If not specified, the method will automatically select the GPU if available, otherwise it will use 
				the CPU.
			- **include_mask_type** <ParameterTypeBadge>bool</ParameterTypeBadge> <DefaultParameterBadge value='false'/>\
				If `true`, the type of the masked entity will be included in the mask token. 
				For example, if the mask token is `"[MASKED]"` and a person's name is masked, 
				it will be replaced with `"[MASKED_PERSON]"`. Default is `false`.
			- **include_mask_counter** <ParameterTypeBadge>bool</ParameterTypeBadge> <DefaultParameterBadge value='false'/>\
				If `true`, a unique identifier will be added to each entity's mask token, 
				singling out that particular entity in the anonymized text. <ins>In order for this 
				parameter to have an effect, `include_mask_type` must also be set to `true`</ins>.
				For example, if the mask token is `"[MASKED]"` and two different people are masked, they 
				will be replaced with `"[MASKED_PERSON_0]"` and `"[MASKED_PERSON_1]"` respectively. This 
				is useful to preserve relationships between multiple occurrences of the same entity in the 
				anonymized text, which would otherwise be lost. Default is `false`.
			- **disable_logging** <ParameterTypeBadge>bool</ParameterTypeBadge> <OptionalParameterBadge />\
				If set to `True`, disables inference metric logging for this call. By default, logging is enabled.
		</APIDescriptionSection>

		<APIDescriptionSection title='Response'>
			A `list[str]`, one per input text. Each string is a version of the corresponding input
			string in which all detected Personally Identifiable Information (PII) have been replaced
			with a placeholder. 
		</APIDescriptionSection>
