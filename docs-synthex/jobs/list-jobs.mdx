<div className='responsive-container'>
	<div className='text-container'>
        # List jobs

        <InfoBox type='warning'>
            This method is only available to users who have signed up on the 
            <a href={PLATFORM_BASE_URL} rel='noreferrer' target='_blank'>Tanaos platform</a> and generated
            an API Key. If you are using Synthex without an API Key, this method will return an authentication
			error.
        </InfoBox>

        Retrieves all jobs that have been executed by the user whose API key is used to authenticate
        the request. Both jobs generated through Synthex and those generated on 
        the <a href={PLATFORM_BASE_URL} rel='noreferrer' target='_blank'>Tanaos platform</a> are included.

		<APIDescriptionSection title='Arguments'>
            - **limit** <ParameterTypeBadge>int</ParameterTypeBadge> <OptionalParameterBadge>optional</OptionalParameterBadge>\
            The maximum number of jobs to retrieve. If not specified, the default value is 10.
            - **offset** <ParameterTypeBadge>int</ParameterTypeBadge> <OptionalParameterBadge>optional</OptionalParameterBadge>\
            The number of jobs to skip before starting to collect the result set. If not specified, the default value is 0.
		</APIDescriptionSection>

		<APIDescriptionSection title='Response'>
            - **total** <ParameterTypeBadge>int</ParameterTypeBadge>\
            The total number of jobs that have been executed by the user.
            - **jobs** <ParameterTypeBadge>list[JobResponseModel]</ParameterTypeBadge>\
            A list of jobs that have been executed by the user. Each job is represented as a `JobResponseModel` object 
            with the following fields:
                - **id** <ParameterTypeBadge>str</ParameterTypeBadge>\
                The job's unique identifier.
                - **name** <ParameterTypeBadge>str</ParameterTypeBadge>\
                The name of the job.
                - **description** <ParameterTypeBadge>str</ParameterTypeBadge>\
                The description of the job.
                - **datapoint_num** <ParameterTypeBadge>int</ParameterTypeBadge>\
                The number of datapoints in the job.
                - **output_domain** <ParameterTypeBadge>str</ParameterTypeBadge>\
                A longer job description, which specifies the kind of job, constraints to its values and other characteristics.
                - **status** <ParameterTypeBadge>JobStatus</ParameterTypeBadge>\
                An object indicating the job status. Possible values are:
                    - `JobStatus.ON_HOLD`: the job is on hold and waiting for user action. This can only occur for jobs
                    generated on the Tanaos platform.
                    - `JobStatus.IN_PROGRESS`: the job is currently being executed.
                    - `JobStatus.COMPLETED`: the job has been successfully executed
                    - `JobStatus.FAILED`: the job has failed.
                - **created_at** <ParameterTypeBadge>datetime</ParameterTypeBadge>\
                The date and time when the job was created.
		</APIDescriptionSection>
