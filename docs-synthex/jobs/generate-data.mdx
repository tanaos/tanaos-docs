        # Generate data

		Generate a dataset with the specified schema, examples and requirements. The model will
		learn from the provided examples and generate a dataset that matches the specified schema.

		<APIDescriptionSection title='Arguments'>
			- **schema_definition** <ParameterTypeBadge>dict</ParameterTypeBadge>\
				A dictionary which specifies the output dataset's schema. It must have the 
				following format:
				```
				{
				    "<col_1_name>": {"type": "<col_1_type>"},
				    "<col_2_name>": {"type": "<col_2_type>"},
				    ...
				    "<col_n_name>": {"type": "<col_n_type>"}
				}
				```
				where the possible values of `<col_n_type>` are `integer`, `float` and `string`.
			- **examples** <ParameterTypeBadge>list[dict]</ParameterTypeBadge>\
				A list of dictionaries, which specifies a few (3 to 5 are enough) sample datapoints 
				that will help the data generation model understand what the output data should 
				look like. <u>Each dictionary must follow the schema specified in 
				the</u> `schema_definition` parameter, or an exception will be raised.
			- **requirements** <ParameterTypeBadge>list[str]</ParameterTypeBadge>\
				A list of strings, where each string specifies a requirement or constraint for the 
				job. <u>It must be an empty list if no specific requirements are present</u>.
			- **output_path** <ParameterTypeBadge>str</ParameterTypeBadge>\
				A string which specifies the path where the output dataset will be generated. 
				It does not need to specify a file name, as this will be added automatically if 
				one is not provided. If a file name is specified, <u>its extension must be 
				consistent with the</u> `output_type` parameter. If this is the case, the provided 
				`output_path` is used in its entirety. Otherwise, the provided extension is replaced 
				with one that is consistent with `output_type`.
			- **number_of_samples** <ParameterTypeBadge>int</ParameterTypeBadge>\
				An integer which specifies the number of datapoints that the model should generate. 
				The maximum number of datapoints you can generate with a single job depends on 
				whether you are on a free or paid plan.
			- **output_type** <ParameterTypeBadge>str</ParameterTypeBadge>\
				A string which specifies the format of the output dataset. Only `"csv"` 
				(meaning a .csv file will be generated) is supported at this time, but we will 
				soon add more options.
		</APIDescriptionSection>

		<APIDescriptionSection title='Response'>
			A <Link to={SYNTHEX_ACTION_RESULT_URL}>Action Result object</Link>.
		</APIDescriptionSection>
